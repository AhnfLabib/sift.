# sift. v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build sift. v1 — a single-user personal finance PWA with fast expense logging (web form + in-app chat + WhatsApp), a budget dashboard, monthly bill reminders, and on-demand Google Sheets export.

**Architecture:** Next.js 15 App Router (TypeScript, Tailwind v4) on Vercel; Supabase Postgres with RLS as the only source of truth; pure-TypeScript domain libraries (parser, money/date helpers, reminder engine) that are unit-tested without any network; thin API routes for the Twilio webhook, cron, and Sheets export using `fetch` against provider REST APIs (no heavy SDKs).

**Tech Stack:** Next.js 15, React 19, Tailwind CSS v4, `@supabase/supabase-js` + `@supabase/ssr`, Vitest, `google-auth-library` (JWT only), Resend REST API, Twilio REST API (WhatsApp sandbox).

**Specs this plan implements:**
- `docs/superpowers/specs/2026-07-02-sift-plan-sanity-check.md` (revised data model, parser grammar, scope cuts, resolved decisions)
- `docs/design/2026-07-02-frontend-design.md` + `docs/design/mockups/dashboard.html` (visual contract)

## Global Constraints

- **Money is integer cents everywhere** (`amount_cents`, `limit_cents`). Never float dollars. Display via `formatCents`.
- **USD only.** No currency column, no locale switching.
- **Single user.** Public signup is disabled operationally; middleware additionally rejects any authenticated user whose email ≠ `ALLOWED_EMAIL` env var (belt and suspenders).
- **RLS on every table**, policy `user_id = auth.uid()` (profiles: `id = auth.uid()`; reminders_log via owning bill). The service-role key is used ONLY in the Twilio webhook route and the cron route; it must never be imported into client components or page components.
- **No secrets in the repo or the client bundle.** Only `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY` may be `NEXT_PUBLIC_`.
- **Timezone rules:** a transaction's `date` is the calendar date in the user's `profiles.timezone` at entry time; the reminder cron computes "today" in the user's timezone. Helper: `todayInTz(tz)`.
- **Bills recur monthly only** (`recurrence = 'monthly'`); due day clamps to the last day of short months (`clampDueDay`).
- **Transaction `source` enum is exactly `'web' | 'chat' | 'sms'`** (WhatsApp arrives as `'sms'`). Reminder `channel` enum is `'email' | 'whatsapp' | 'sms'`.
- **Uncategorized = `category_id IS NULL`.** "Other" is a real category the user picks deliberately.
- **Preset categories (seed exactly these 9, in this order):** Rent, Food, Transit, Utilities, Subscriptions, Entertainment, Health, Shopping, Other.
- **Design tokens are law** (from `docs/design/2026-07-02-frontend-design.md`): `--ink #23271F`, `--blotter #E5E4DA`, `--page #FBF6E9`, `--banker #1E3A2D`, `--debit #A93F2E`, `--feint #9FB6C4`. Fonts: Besley (display), Public Sans (body/UI), IBM Plex Mono (all amounts/dates/parser echoes). Material rule: paper for the record (ledger card — matte, opaque), glass for the tool layer (top bar, quick-add/chat panels, FAB), blotter for everything else. The mockup `docs/design/mockups/dashboard.html` is the canonical rendering of the dashboard; match it.
- **Copy voice:** sentence case, plain verbs ("Log expense"), confirmations echo the parse ("Logged $14.00 — Chipotle — Food."), errors say what happened and what to do next. No exclamation marks in errors.
- **Quality floor:** responsive to 360px, visible keyboard focus, `prefers-reduced-motion` respected, hit targets ≥44px on mobile.
- **YAGNI (explicitly cut from v1):** weekly/custom recurrence, nightly Sheets sync, push notifications, multi-currency, SMS numbered-reply categorization, budget auto-rollover (a "Copy last month" button instead).
- **Testing:** Vitest. Pure domain logic (parser, helpers, reminder engine, webhook signature, sheets payload) is TDD with the test cases given in each task. UI and DB glue are verified by `npm run build` + lint; do not write tests that merely mock Supabase and assert the mock.
- Run `npm run test` and `npm run build` before each task's final commit. Commit frequently with `feat:`/`fix:`/`chore:` prefixes.

### Task 3: Expense parser (TDD)

**Files:**
- Create: `src/lib/parser/parse.ts`, `src/lib/parser/parse.test.ts`

**Interfaces:**
- Produces:

```ts
export interface KeywordEntry { keyword: string; categoryId: string }
export interface ParsedExpense {
  amountCents: number;
  merchant: string;            // title-cased; "Unknown" when only an amount was given
  categoryId: string | null;   // from keyword match, else null
  matchedKeyword: string | null;
}
export function parseExpense(raw: string, keywords: KeywordEntry[]): ParsedExpense | null;
```

**Grammar (from the spec — implement exactly):**
1. Tokenize on whitespace after trimming. Amount candidates match `/^\$?\d{1,6}(?:,\d{3})*(?:\.\d{1,2})?$/` (strip commas); a bare number followed by the word `bucks`, `dollars`, or `usd` counts as dollar-marked (and that word is consumed).
2. Pick the amount: (a) first `$`-prefixed or dollar-word-marked candidate; else (b) if exactly one candidate, use it; else (c) among decimal-bearing candidates, the largest (a strict maximum — a tie means ambiguous → return null); else (d) multiple plain integers → return null.
3. `amountCents = Math.round(value * 100)`; reject zero and anything over $100,000 (return null).
4. Merchant: remaining tokens minus filler words `for on at i spent paid` (case-insensitive, whole-token), joined, title-cased per word. Empty → `"Unknown"`.
5. Category: case-insensitive match of each keyword against the merchant text on word boundaries; longest keyword wins; ties broken by first in input order.
